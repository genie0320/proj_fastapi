# 흉부 X-Ray AI 진단 서비스 요구사항 정의서 (SRS)

## 1. 개요
본 문서는 사내 의료인 및 개발, 연구 실무진을 위한 **흉부 X-Ray AI 진단 서비스**의 사용자 관리(User Management) 기능 요구사항 및 비기능 요구사항을 정의합니다.

---

새로 전달해주신 **AI 모델 예측(`PRED`)** 기능 및 비기능 요구사항을 반영하여, 전체 시스템의 요구사항 정의서(SRS)를 완벽한 마크다운(.md) 형태로 업데이트했습니다.

특히 `NFR-PRED-001(AI 모델 평가 기준)`에서 지적된 **FN(Falsely Negative, 위음성)의 위험성과 재현율(Recall) 기준**은 흉부 X-Ray 의료 진단 시스템의 안전성을 좌우하는 핵심 성능 한계점(SPOF)이므로, 명세서 내에 수식 및 제약 조건이 명확히 드러나도록 구조화했습니다.

## 1. 사용자 관리 (User Management)

### REQ-USER-001: 회원가입

* **구분:** 기능 요구사항
* **기능 상세:** 사내 구성원은 회원가입을 통해 흉부 X-Ray AI 진단 서비스를 이용할 수 있어야 한다.
* **필수 입력 항목:** 이메일, 비밀번호, 이름, 부서 (연구, 의료, 개발), 성별 (M / F), 휴대폰 번호

### REQ-USER-002: 로그인

* **구분:** 기능 요구사항
* **기능 상세:** 회원가입을 완료한 사내 개발진, 의료 실무진, 연구진은 이메일과 비밀번호를 입력하여 로그인을 진행한 후 서비스를 이용할 수 있다.

### REQ-USER-003: 로그아웃

* **구분:** 기능 요구사항
* **기능 상세:** 로그인 유저는 상시 헤더에 노출되는 로그아웃 버튼을 통해 로그아웃을 진행할 수 있으며, 로그아웃 시 로그인 페이지로 전환된다.

### REQ-USER-004: 회원 목록 조회

* **구분:** 기능 요구사항
* **기능 상세:** 관리자 권한(`Admin`) 유저는 회원관리 메뉴에 접속하여 모든 회원을 목록으로 조회할 수 있다.
* **검색/필터:** 이메일 혹은 이름으로 검색이 가능하며, 필터 드롭다운으로 부서별 조회가 가능하다.
* **목록 노출 항목:** 고유 ID, 이메일, 이름, 부서, 성별, 휴대폰 번호, 계정 활성화 여부



### REQ-USER-005: 회원 권한 변경

* **구분:** 기능 요구사항
* **기능 상세:** 관리자 권한(`Admin`) 유저는 목록에서 대상자를 선택하고 상단의 버튼을 클릭하여 회원의 권한을 변경할 수 있다.
* **권한 등급 세부 정의 (비고):**
* **대기자:** 마이페이지 외 모든 서비스 접근 불가
* **스태프:** 내부 직원, 흉부 X-ray 관련 모든 읽기, 쓰기, 수정 작업 가능
* **어드민:** 시스템 관리자, 모든 항목에 대한 데이터 엑세스 가능



### REQ-USER-006: 마이페이지 조회

* **구분:** 기능 요구사항
* **기능 상세:** 모든 로그인 유저는 마이페이지에서 본인의 정보(이름, 이메일, 부서, 성별, 휴대폰 번호, 권한)를 확인할 수 있다.

### REQ-USER-007: 회원 정보 수정

* **구분:** 기능 요구사항
* **기능 상세:** 모든 로그인 유저는 마이페이지에서 본인의 정보를 수정할 수 있다. (Partial / PATCH 방식 처리)
* **수정 가능 항목:** 부서, 휴대폰번호

### REQ-USER-008: 비밀번호 변경

* **구분:** 기능 요구사항
* **기능 상세:** 기존 비밀번호 일치 여부를 검증한 후 새로운 비밀번호를 적용한다.
* **입력 항목:** 기존 비밀번호, 새로운 비밀번호

### REQ-USER-009: 회원 탈퇴

* **구분:** 기능 요구사항
* **기능 상세:** 모든 로그인 유저는 마이페이지에서 회원탈퇴를 진행할 수 있으며, 탈퇴 시 Database에서 회원 관련 정보는 모두 즉시 삭제(Hard Delete)한다.

## 2. 환자 관리 (Patient Management)

### REQ-PTNT-001: 환자 정보 등록

* **구분:** 기능 요구사항
* **기능 상세:** 사내 의료인 유저는 환자 관리 메뉴에서 모달을 통해 환자 정보를 시스템 상에 등록할 수 있다.
* **필수 입력 항목:** 이름, 나이, 성별, 연락처 (휴대폰 번호)

### REQ-PTNT-002: 환자 목록 조회

* **구분:** 기능 요구사항
* **기능 상세:** 로그인 된 사내 구성원은 환자 관리 메뉴에서 등록된 환자 정보를 목록으로 조회할 수 있다.
* **검색/필터:** 이름 기준 검색, 성별 및 나이범위 기준 필터링 제공
* **노출 항목:** 환자 고유 ID, 이름, 나이, 성별, 연락처, 생성일시, 수정일시



### REQ-PTNT-003: 환자 정보 상세 조회

* **구분:** 기능 요구사항
* **기능 상세:** 환자 목록 우측의 상세보기 버튼을 통해 상세 페이지로 이동하여 환자의 세부 정보(이름, 성별, 연락처, 나이)를 확인한다.

### REQ-PTNT-004: 환자 정보 수정

* **구분:** 기능 요구사항
* **기능 상세:** 환자 정보 상세보기 페이지 내 모달을 통해 정보를 수정한다. (수정 가능 항목: 이름, 연락처)

### REQ-PTNT-005: 환자 정보 삭제

* **구분:** 기능 요구사항
* **기능 상세:** 삭제 버튼 클릭 시 안내 팝업이 노출되며, 최종 확인 시 해당 환자와 연관된 진료기록, X-Ray 이미지도 함께 영구 삭제(Cascading Hard Delete)한다.

## 3. 진료기록 관리 (Medical Records)

### REQ-MDR-001: 진료기록 등록

* **구분:** 기능 요구사항
* **기능 상세:** 사내 의료인 유저는 환자 상세 페이지에서 진료 기록 입력 페이지로 이동하여 환자의 X-Ray 사진을 포함한 진료기록을 등록한다.
* X-Ray 이미지 업로드 시 미리보기(Preview)를 제공한다.
* **필수 입력 항목:** 환자 정보(고유 ID), 진료 정보(진료 차트 넘버, 진료된 증상, 촬영된 흉부 X-ray 이미지)


* **비고:** 이미지 저장 경로는 서버가 실행되는 환경의 **로컬 저장소**에 저장한다.

### REQ-MDR-002: 진료기록 목록 조회

* **특이 명세:** 증상 필드 내용이 **100자 초과 시 ‘…’ 생략 형태로 노출**한다.
* **노출 항목:** 진료 기록 ID, 진료 차트 넘버, 증상, 생성일시



### REQ-MDR-003: 진료기록 상세 조회

* **구분:** 기능 요구사항
* **기능 상세:** 진료기록 항목의 상세보기 버튼을 클릭하여 상세 페이지로 이동하며 진료 기록 ID, 차트 넘버, 증상, 흉부 X-Ray 이미지, 생성일시를 조회한다.

## 4. AI 모델 예측 및 결과 관리 (AI Prediction Management) 

### REQ-PRED-001: AI 모델 활용 폐렴 예측

* **구분:** 기능 요구사항
* **기능 요약:** 등록된 흉부 X-Ray 이미지를 기반으로 한 폐렴 AI 예측 수행
* **기능 상세:**
* 사내 의료인, 개발팀, 연구자 권한을 가진 유저는 진료기록 상세 페이지에서 'AI 예측 결과보기' 버튼을 클릭하여 폐렴 예측 모델의 추론 결과를 확인할 수 있어야 한다.
* **데이터 연동:** 폐렴 예측 알고리즘에 필요한 X-ray 이미지는 진료기록 저장 시 업로드했던 원본 이미지를 그대로 활용한다.
* **추론 캐싱 정책 (Caching):** 만약 해당 진료기록에 대해 동일한 버전의 AI 모델로 기저장된 예측 결과 데이터가 존재한다면, 중복 추론을 거치지 않고 데이터베이스에 저장되어 있던 기존 데이터를 즉시 반환하여 응답 속도를 최적화한다.
* **UI 배치:** 예측 결과는 진료기록 상세 페이지 내 'AI 예측 결과 섹션'에서 별도 화면 이동 없이 바로 확인할 수 있도록 구성한다.
* **결과 반환 데이터:**
* 폐렴 여부 (`True` / `False`)
* 신뢰도 (`Confidence`, 예측 확률 값)
* 히트맵 이미지 URL (`Hitmap Image URL` / 선택 사항 - 병변 시각화 활성화 시 사용)





### REQ-PRED-002: AI 모델 활용 폐렴 예측 결과 조회

* **구분:** 기능 요구사항
* **기능 요약:** 시계열적 AI 예측 히스토리 이력 목록 조회
* **기능 상세:**
* 사내 의료인, 개발팀, 연구자 권한을 가진 유저는 진료기록 상세 페이지 내의 AI 예측 결과 섹션에서, 해당 환자의 흉부 X-Ray 사진과 함께 폐렴 예측 모델을 활용해 수행되었던 예측 결과 이력을 목록 형태로 조회할 수 있어야 한다.
* **목록 노출 항목:** 고유 ID, 폐렴 여부, Confidence, Hitmap Image URL, 예측 수행 일시, 사용한 모델 버전 명칭

## 5. 비기능 요구사항 (Non-Functional Requirements)

### NFR-USER-001: 인증 / 인가

* 로그인 시 `JWT`를 발급하며 API 인가에 사용한다. 만료 주기는 액세스 토큰 **30분**, 리프레시 토큰 **7일**로 한다.
* 리프레시 토큰은 클라이언트 접근이 불가한 **`http_only` 쿠키**로 보관하며, JWT 페이로드에는 최소 식별 정보인 `user_id`만 저장한다.

### NFR-USER-002: 비밀번호 입력 보안

* 모든 비밀번호 인풋 필드는 마스킹 처리하되, '비밀번호 보기' 아이콘 토글을 제공한다.

### NFR-PRED-001: AI 모델 평가 및 품질 기준 `[★신규 추가]`

* **목적:** 의료 데이터 진단의 오진율을 제어하기 위해 인공지능 모델의 성능 하한선을 강제 정의한다.
* **매트릭스 분류 정의:**
* **TP (True Positive):** 실제 폐렴 환자를 폐렴으로 올바르게 진단한 경우
* **FP (False Positive):** 정상인이지만 폐렴 환자로 오진한 경우 (과잉 진단)
* **FN (False Negative):** **실제 폐렴 환자인데 정상으로 오진한 경우 (⚠️ 임상적으로 가장 위험한 단일 장애점)**
* **TN (True Negative):** 정상인을 정상으로 올바르게 진단한 경우


* **성능 제어 평가지표 기준:**
* **재현율 (Recall / 민감도) $\ge$ 0.90 ~ 0.95 (최소 0.90 필수 보장):** * *정의:* 실제 폐렴 환자 중 모델이 놓치지 않고 잡아낸 비율 (위음성 방지)
* *계산식:* $\frac{TP}{TP + FN}$


* **정확도 (Accuracy) $\ge$ 0.80 ~ 0.90 (보조 지표 활용):**
* *정의:* 전체 검사 표본 중 올바른 진단을 내린 비율
* *계산식:* $\frac{TP + TN}{\text{전체 표본 수}}$





### NFR-USER-003 / NFR-PTNT-001 / NFR-MDR-001 / NFR-PRED-002: API 응답 성능

* 사용자 관리, 환자 관리, 진료기록 관리를 비롯하여 신규 추가된 **AI 모델 추론 처리 및 이력 조회 API(`PRED`)를 포함한 모든 시스템 API는 부하 상황을 제외하고 최대 3초 이내에 로직을 처리하고 최종 응답을 반환**해야 한다.
